REST(Representational State Transfer)

- 스프링에서 REST 서비스를 설계할 때는 2가지 가능성을 고려

1) Application 자체를 REST 서비스로 발행
2) 애플리케이션에서 쓸 데이터를 서드파티 REST 서비스에서 가져오는 것


@RequestMapping, @PathVariable

- 서드 파티 라이브러리 사용(ROME)
